🔒 Privacy & data use
Your files never leave your device. All OCR, PDF reading, narration, and document storage happen locally in your browser.

What may go online (only when you choose):
  • Show Meanings sends Sanskrit words to the Cologne Monier-Williams dictionary API and Google Translate. Disabled by default.
  • Split Sandhi (optional) sends Sanskrit verse text to the University of Hyderabad Heritage segmenter (via your own Cloudflare Worker). Only the verse text is sent — no files. Falls back to an on-device assistant if unreachable. Its per-word meanings send individual words to Google Translate (same as Show Meanings).
  • Cloud OCR and Re-OCR a region (optional) send page images, or just the cropped region you select, to Google Cloud Vision or OCR.space. Disabled by default; requires explicit configuration with your own API key.
  • Drive features connect to Google when you sign in or paste a Drive link.
  • App libraries (PDF reader, OCR engine) load from a public CDN on first launch, then are cached for offline use.
No cookies, no analytics, no tracking. Your voice prefs, Drive credentials, language/text corrections, and saved sandhi splits are stored only in this browser (localStorage).

    ☁️ Cloud OCR Setup

    Cloud OCR significantly improves accuracy on poor-quality scans, photos, and books with underlines or marginalia. The local browser OCR remains a transparent fallback if cloud OCR is unreachable.
    ⚠ Privacy: When enabled, page images are sent to your chosen cloud provider for processing. Files no longer stay only on your device. Each provider has its own data-handling policy — review before enabling.
    How this works: Google Vision blocks direct browser→API calls (CORS). This option routes through a tiny proxy on your own Cloudflare Worker (free tier, 100k requests/day). Your Google API key lives on Cloudflare as an encrypted secret — never exposed in the browser.

    ⚠ Important — set a budget alert. Google Cloud requires billing to be enabled even for the free tier (1,000 calls/month). Set a $1 budget alert in Google Cloud Billing → Budgets so you're notified before any charge.

    📖 Part 1: Get a Google Vision API key (≈ 7 min)
    1. Go to console.cloud.google.com & sign in
    2. Top bar → project picker → "NEW PROJECT" → name it (e.g. indic-reader) → CREATE. Select the new project.
    3. ☰ menu → "Billing" → "LINK A BILLING ACCOUNT" → add a card. Free tier won't charge unless you exceed 1,000 calls/month.
    4. ☰ menu → "Billing" → "Budgets & alerts" → "CREATE BUDGET":
      • Amount: $1
      • Threshold: 50%, 90%, 100% (defaults)
      • Save — this emails you if you would be charged
    5. ☰ menu → "APIs & Services" → "Library" → search "Cloud Vision API" → click → "ENABLE"
    6. ☰ menu → "APIs & Services" → "Credentials" → "+ CREATE CREDENTIALS" → "API key"
    7. A popup shows your new key. Copy it & save somewhere safe.
    8. Click "EDIT API KEY" (popup or in credentials list):
      • API restrictions: select "Restrict key", check only "Cloud Vision API"
      • Application restrictions: "None" (the Worker calls don't preserve a browser referrer)
      • SAVE
    📖 Part 2: Deploy the Cloudflare Worker (≈ 3 min)
    1. Go to dash.cloudflare.com → Workers & Pages
    2. If you have an existing Worker (e.g. indic-ocr-proxy): open it. Otherwise: Create → "Hello World" → name it → Deploy.
    3. Open Worker → Edit code. Select all → delete → paste contents of cloudflare-worker-google-ocr.js → Save and Deploy.
    4. Worker → Settings → Variables and Secrets → add two:
      • GOOGLE_API_KEY = your Google key from Part 1. Mark as Encrypt.
      • ALLOWED_ORIGIN = your app URL, e.g. https://indic-reader.pages.dev
    5. Copy your Worker URL (e.g. https://indic-ocr-proxy.YOUR.workers.dev) and paste below.
    Setup: Get a free key at ocr.space/ocrapi/freekey. Free tier covers 25 000 pages/month. Note: free tier has no Sanskrit model; works well for Gujarati only. For mixed Gujarati+Sanskrit scriptures, use the Cloudflare Worker → Google option above.
    💡 Saving here only stores your credentials. To actually use cloud OCR, switch on the ☁️ Cloud OCR toggle on the main screen.

    🔒 Data privacy

    What stays on your device (always):

    • Every file you open — PDFs, images, DOCX, pasted text
    • OCR results and extracted text
    • Page bookmarks, page selections, and current playback position
    • Theme preference, voice preference, chant mode, auto-scroll, language filter
    • Drive Picker credentials (if you configured them) — stored only in this browser's localStorage
    • Translation cache — built up locally as you tap Show Meanings
    • Manual corrections — your per-word and per-line language overrides, line text corrections (from Re-OCR a region), manual verse references (for vedabase.io links), and saved sandhi splits

    What may go online (only with your action):

    • Show Meanings sends individual Sanskrit words to two third-party services for accurate dictionary lookup. Disabled by default; requires explicit consent. The Cologne University Monier-Williams API (api.c-salt.uni-koeln.de) is queried first as the authoritative source. Google Translate's public endpoint is used as a fallback and to translate English definitions into Gujarati. Each call exposes your IP address and the word being looked up to these services.
    • Cloud OCR (optional, disabled by default; requires explicit consent) sends the entire page image to one of two providers you can configure:
      • Google Cloud Vision via your own Cloudflare Worker — the page image traverses Cloudflare's edge (subject to their data-processing terms) and is sent to Google Cloud Vision. Google's privacy policy applies; per Google's documentation, Vision API requests are not used to improve their models without your explicit consent in the Google Cloud Console settings.
      • OCR.space direct — the page image is sent to ocr.space servers. Per their privacy policy, free-tier requests may be retained for service improvement.
      When Cloud OCR is enabled, files no longer stay only on your device. The processing provider sees your IP address and the page image contents. Recognition results return to your browser only — they are not stored on our side.
    • Split Sandhi (optional, disabled until you use it) sends only the Sanskrit verse text — never any file or image — to the University of Hyderabad Heritage segmenter (sanskrit.uohyd.ac.in), a public academic service, routed through your own Cloudflare Worker. The Worker caches results for 24 hours. If the service is unreachable, the app silently falls back to an on-device rule-based assistant; no text leaves your device in that case. The segmenter sees the verse text and (via the Worker) Cloudflare's edge sees the request. The overlay also shows a per-word meaning in your chosen language (Gujarati/Hindi/English) — this sends each individual word to Google Translate, and tapping a word queries the Cologne Monier-Williams dictionary, both subject to the same consent as Show Meanings below. Your chosen meaning-language is stored locally.
    • Re-OCR a region (optional) sends only the cropped rectangle you select — not the whole page — to your chosen OCR provider (Google Cloud Vision via your Worker, or OCR.space, or processed entirely on-device with local OCR). You also choose the language hint per region. Same provider data-handling as Cloud OCR below.
    • Google Drive Picker/link load — only when you click those buttons. Drive sees the URLs you fetch and (for Picker) your Google account identity.
    • First-launch CDN load — the PDF reader, OCR engine, and DOCX parser load from cdn.jsdelivr.net. After first load they're cached locally and re-used offline. The CDN sees your IP address on first load only.
    • Tesseract language data — Gujarati and Devanagari recognition data is downloaded from tessdata.projectnaptha.com on first OCR. Cached afterward.

    No cookies are set by this app. No analytics, no telemetry, no tracking pixels. We don't operate a server — the app runs entirely in your browser. If you host it elsewhere (e.g. GitHub Pages), that host's standard logs apply.

    ⚠️ Disclaimers

    • OCR accuracy varies with scan quality. Always verify against the original.
    • Inline script-switching is a known OCR limitation: short Sanskrit/Devanagari phrases embedded mid-sentence within Gujarati or Hindi prose are sometimes misread as the surrounding script (e.g. भगवान् काव्यः may appear as માવાનું વાસ્થ્યઃ). This affects all OCR engines — Google, Azure, Tesseract — because text-region detection assigns one dominant script per visual line. Standalone Sanskrit verse lines and standalone Gujarati paragraphs OCR correctly. For inline mixed-script passages, verify against the original page image (Document View).
    • Sandhi-viccheda (compound word splitting) uses the University of Hyderabad Heritage morphological segmenter when online, with an on-device rule-based assistant as fallback. The segmenter gives the ranked best analysis but Sanskrit segmentation has inherent ambiguity — treat results as a study aid and use the built-in split/merge editing to verify. The fallback assistant is heuristic only.
    • Chhand (metre) detection: when a verse matches a known guru/laghu pattern, the exact metre name is shown (rigorous scansion); otherwise it falls back to an approximate syllable-count family. The 📐 Scan view marks each syllable heavy/light by fixed rules. All computed on-device; accuracy depends on clean text (OCR errors shift the marks). Treat it as a study aid.
    • Re-OCR a region and manual language/text overrides let you correct mistakes, but corrections are only as accurate as what you enter or accept. Verify against the original page image.
    • Auto-translation is statistical, not a Sanskrit dictionary. For canonical meanings consult Monier-Williams, Apte, or a trusted teacher.
    • Narration depends on the Text-to-Speech voices installed on your device. Quality varies by platform.
